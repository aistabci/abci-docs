# 24. クラウドストレージアカウントディレクトリの無効化

無効化機能は、即座にデータ公開を停止するための機能です。 
たとえば、インシデントが発生した場合等に利用可能です。  

無効化したCSADは、利用者ポータルからは有効化できません。  
有効化をご希望の場合は、利用責任者または利用管理者より、<abci3-qa@abci.ai> へメールで依頼してください。  

無効化した場合、該当クラウドストレージアカウントディレクトリに対する一連の操作が不可となります。

利用グループ詳細画面の右下にある「グループ内CSアカウントディレクトリ」の[すべて表示]をクリックします。

![[[Screenshot]]](img/18_A.png)

無効化対象の[CASD名]をクリックします。

![[[Screenshot]]](img/18_B.png)

クラウドストレージアカウントディレクトリの詳細画面に遷移し、右の[無効化]をクリックします。

![[[Screenshot]]](img/18_C.png)

注意事項を確認し、[次へ]をクリックします。

![[[Screenshot]]](img/18_D.png)

無効化の受付完了画面が表示されます。

![[[Screenshot]]](img/18_E.png)

以下の関係者に通知メールが送信されます。
- 申請者
- 該当グループの責任者
- 該当グループの管理者
- 対象クラウドストレージアカウントディレクトリの所有者

「クラウドストレージアカウントディレクトリ無効化」が完了すると、申請者に完了通知のメールが送信されます。

!!! note  
    ・ACL（アクセス制御リスト）が初期化され、これまでに設定されていたすべてのアクセス権が削除されます。  
    ・対象のクラウドストレージアカウント内に存在するすべてのバケットに対して、全IPアドレスからのアクセスを拒否するバケットポリシーが適用されます。  
    ・無効化が完了すると、無効化前に発行されていたシークレットキーは使用できなくなります。

!!! note  
    ・無効化されたクラウドストレージアカウントディレクトリは、再度有効化することが可能です。  
    ・クラウドストレージアカウントディレクトリの有効化を希望される場合や、ACL（アクセス制御リスト）の再設定手順について確認されたい場合は、利用責任者または利用管理者を通じて、以下のメールアドレスまでご連絡ください。  
    <abci3-qa@abci.ai>   
    ・無効化直前にシステムが取得した、ACL（アクセス制御リスト）のバックアップファイルは、利用者が指定されたディレクトリへ提供可能です。  
    ・メールでのご連絡方法については、お問い合わせ（<https://abci.ai/ja/how_to_use/user_support.html>）をご参照ください。
!!! note          
    クラウドストレージアカウントディレクトリを有効化すると、シークレットキーが再生成されます。これにより、以前のシークレットキーは使用できなくなります。
