# 24. クラウドストレージアカウントディレクトリの無効化

無効化機能は、インシデントが発生した場合に即座にデータ公開を停止するための機能です。  
無効化した場合、該当クラウドストレージアカウントディレクトリに対する一連の操作が不可となります。

利用グループ詳細画面の右下にある「グループ内CSアカウントディレクトリ」の[すべて表示]をクリックします。

![[[Screenshot]]](img/18_A.png)

無効化対象の[CASD名]をクリックします。

![[[Screenshot]]](img/18_B.png)

クラウドストレージアカウントディレクトリの詳細画面に遷移し、右の[無効化]をクリックします。

![[[Screenshot]]](img/18_C.png)

注意事項を確認し、[次へ]をクリックします。

![[[Screenshot]]](img/18_D.png)

無効化の受付完了画面が表示されます。

![[[Screenshot]]](img/18_E.png)

以下の関係者に通知メールが送信されます。
- 申請者
- 該当グループの責任者
- 該当グループの管理者
- 対象クラウドストレージアカウントディレクトリの所有者

「クラウドストレージアカウントディレクトリ無効化」が終了したかどうかについては、 <abci3-qa@abci.ai> へメールで質問してください。

!!! note
    クラウドストレージを無効化すると、ACLが初期化され、結果として過去に設定されたACLがすべて消え、以前のアクセス権の設定が消えます。
    また、該当クラウドストレージアカウントディレクトリに存在する全てのバケットに全IPアドレスからのアクセスを拒否するバケットポリシーが適用されます。
    無効化が完了すると、無効化前に設定されていたシークレットキーは使用できなくなります。

!!! note
    無効化したクラウドストレージアカウントディレクトリを有効化することが可能です。
    有効化を希望される場合、ACL再設定の手順を確認したい場合は、利用責任者または利用管理者より、<abci3-qa@abci.ai> へメールで依頼してください。
    また、システムにより無効化直前に aws s3api get-bucket-acl コマンドで取得したバケット ACL のバックアップファイルについては、有効化の際に、利用者が指定するディレクトリへ提供することが可能です。
     <abci3-qa@abci.ai> へのご連絡については [お問い合わせ](https://docs.abci.ai/v3/ja/contact/) をご参照ください。

!!! note          
    クラウドストレージアカウントディレクトリを有効化すると、シークレットキーが再生成されます。これにより、以前のシークレットキーは使用できなくなります。
